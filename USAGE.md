# 使用指南

## 快速开始

### 1. 安装 Playwright 浏览器

首次使用前，需要安装 Playwright 的 Chromium 浏览器：

```bash
npx playwright install chromium
```

### 2. 运行同步

使用以下命令之一开始同步：

#### 使用 Claude Code Skill（最简单）

```bash
/xz          # 增量更新（默认）
/xz update   # 增量更新（明确指定）
/xz full     # 全量下载
```

#### 使用 npm 命令

```bash
npm run sync:update  # 增量更新
npm run sync:full    # 全量下载
```

### 3. 首次使用流程

1. 运行命令后，浏览器会自动打开
2. 如果未登录知识星球，请在浏览器中完成登录：
   - 可以使用手机扫码登录
   - 或者输入手机号验证码登录
3. 登录成功后，工具会自动开始抓取内容
4. 等待抓取完成（控制台会显示进度）
5. 检查 `output/` 目录中生成的 Markdown 文件

## 同步模式详解

### 全量下载 (`full`)

**适用场景**：首次使用

**执行操作**：
- 滚动加载星球中的所有帖子
- 筛选出包含许哲内容的帖子（他发布的帖子或他的评论）
- 下载所有相关的图片和附件
- 生成 Markdown 文件
- 更新同步状态

**注意事项**：
- 可能需要较长时间（取决于内容数量）
- 建议在网络稳定的环境下运行
- 浏览器会保持打开状态，方便观察进度

**命令**：
```bash
/xz full
# 或
npm run sync:full
```

### 增量更新 (`update`)

**适用场景**：日常使用

**执行操作**：
- 只检查最新的 20 个帖子
- 对比同步状态，识别：
  - 新发布的帖子（包含许哲内容）
  - 有新评论的帖子（许哲的新评论）
- 只下载有变化的帖子
- 更新同步状态

**优势**：
- 速度快
- 节省资源
- 适合定期运行

**命令**：
```bash
/xz
# 或
/xz update
# 或
npm run sync:update
```

### 分批下载 (`full [数量]`)

**适用场景**：时间有限，想分多次完成下载

**执行操作**：
- 每次处理指定数量的未处理帖子
- 已处理的帖子会被跳过
- 多次运行自动继续下载剩余内容
- 灵活控制每次下载的节奏

**优势**：
- 时间可控
- 随时可停止
- 渐进式下载
- 资源占用小

**命令**：
```bash
# 每次下载 10 个
/xz full 10
# 或简写
/xz 10

# 每次下载 20 个
/xz full 20
# 或简写
/xz 20

# 不限制数量（全部下载）
/xz full
```

**示例工作流**：
```bash
# 第1次：下载前10个
/xz 10

# 第2次：继续下载接下来10个
/xz 10

# 第3次：下载剩余所有
/xz full
```

详细说明请查看 [BATCH_DOWNLOAD.md](BATCH_DOWNLOAD.md)

### 断点续传（全量下载专属）

**功能说明**：

全量下载过程中如果遇到中断（网络断开、手动停止、程序崩溃等），可以从上次中断的位置继续下载，而不需要重新开始。

**工作原理**：

1. **自动保存进度**
   - 下载过程中每发现更早的内容时自动保存进度
   - 记录已下载到的最早时间戳
   - 记录已下载的总数

2. **自动恢复**
   - 再次运行 `/xz full` 时自动检测未完成的下载
   - 从上次的最早时间点继续往前下载
   - 已下载的内容不会重复处理

3. **进度显示**
   ```
   ⚠️  检测到未完成的下载
   📍 将从上次中断位置继续（最早时间: 2026-01-15T10:00:00Z）
   📊 已下载: 150 个帖子

   💾 保存进度: 已下载到 2026-01-10T08:30:00Z, 共 200 个帖子
   ```

**使用示例**：

```bash
# 第一次运行（假设中途中断了）
/xz full
# ... 下载了一部分后中断 ...

# 再次运行，会自动继续
/xz full
# ⚠️  检测到未完成的下载
# 📍 将从上次中断位置继续...
```

**何时使用**：

- ✅ 网络不稳定时
- ✅ 内容量很大，担心中途出问题
- ✅ 需要分多次完成下载
- ✅ 手动停止后想继续

**清除进度重新开始**：

如果想放弃当前进度，重新开始全量下载：

```bash
# 删除进度信息（保留已下载的帖子记录）
rm .xz/sync_state.json

# 重新开始
/xz full
```

## 输出文件

### 文件命名规则

Markdown 文件名格式：`YYYY-MM-DD_HHmm_标题前20字符.md`

例如：
- `2026-01-28_1030_如何提高学习效率.md`
- `2026-01-27_1500_关于投资的思考.md`

### Markdown 文件结构

每个文件包含：

1. **Frontmatter（元数据）**
   ```yaml
   ---
   title: "帖子标题"
   author: 许哲
   date: 2026-01-28T10:00:00Z
   topic_id: xxx
   source: https://wx.zsxq.com/...
   type: talk
   likes: 10
   comments: 5
   ---
   ```

2. **标题和基本信息**
   ```markdown
   # 帖子标题

   **作者**: 许哲
   **发布时间**: 2026-01-28 10:00
   **点赞数**: 10
   ```

3. **正文内容**
   - 文字内容
   - 图片（本地引用）
   - 附件链接

4. **评论区**
   - 所有评论和回复
   - 包含许哲的评论会特别标注

### 图片和附件

- **图片**：保存在 `output/images/` 目录
  - 文件名格式：`{topic_id}_{序号}.{扩展名}`
  - Markdown 中使用相对路径引用：`![图片](images/xxx.jpg)`

- **附件**：保存在 `output/files/` 目录
  - 保留原始文件名
  - Markdown 中以链接形式展示

## 同步状态管理

### 状态文件：`.xz/sync_state.json`

记录内容：
```json
{
  "last_sync_time": "2026-01-28T10:00:00Z",
  "synced_topics": {
    "topic_123": {
      "last_updated": "2026-01-28T10:00:00Z",
      "comment_count": 5
    }
  },
  "xu_zhe_user_id": "xxx"
}
```

### 增量更新的判断逻辑

工具会检查：
1. **新帖子**：`topic_id` 不在 `synced_topics` 中
2. **新评论**：当前评论数 > 记录的 `comment_count`

如果满足任一条件，会重新下载该帖子的完整内容。

## 配置文件

### 配置文件：`.xz/config.json`

```json
{
  "group_id": "28888825825151",
  "group_name": "天上不会掉馅饼",
  "target_author": "许哲",
  "output_dir": "/Users/tony/dev/personal/xz/output",
  "api_base_url": "https://api.zsxq.com/v2"
}
```

### 修改目标作者

如果要下载其他作者的内容，修改 `target_author` 字段：

```json
{
  "target_author": "其他作者名"
}
```

然后删除 `.xz/sync_state.json` 并重新运行全量下载。

## 常见问题

### 1. 浏览器无法打开

**问题**：运行命令后浏览器没有打开

**解决**：
```bash
# 安装 Playwright 浏览器
npx playwright install chromium
```

### 2. 登录后页面空白

**问题**：登录成功后看不到内容

**解决**：
- 等待几秒钟，页面可能还在加载
- 手动刷新页面
- 检查星球 ID 是否正确

### 3. 图片下载失败

**问题**：部分图片显示为原始 URL

**原因**：
- 图片可能有访问限制
- 网络连接问题

**影响**：
- Markdown 文件仍然包含图片链接
- 可以在线查看（需要登录）

### 4. 找不到许哲的内容

**问题**：运行后显示 0 个帖子

**检查**：
1. 确认作者名称是否正确（在配置文件中）
2. 确认星球 ID 是否正确
3. 确认已登录正确的账号

### 5. 增量更新没有检测到新内容

**问题**：明明有新评论，但没有下载

**原因**：
- 新评论可能不在最新 20 个帖子中
- 需要运行全量下载

**解决**：
```bash
/xz full
```

### 6. 浏览器一直不关闭

**说明**：
- 这是预期行为
- 方便用户检查抓取结果
- 可以手动关闭浏览器

### 7. 全量下载中途中断了怎么办？

**答案**：直接再次运行 `/xz full` 即可

**说明**：
- 工具会自动检测未完成的下载
- 从上次中断的位置继续下载
- 不会重复下载已保存的内容
- 支持多次中断和恢复

**示例**：
```bash
# 即使中断了，再次运行相同命令
/xz full
# ⚠️  检测到未完成的下载
# 📍 将从上次中断位置继续...
```

### 8. 如何清除断点续传进度？

**问题**：想重新开始全量下载，不要断点续传

**解决**：
```bash
# 删除状态文件
rm .xz/sync_state.json

# 重新开始全量下载
/xz full
```

### 9. 重复运行会覆盖文件吗？

**答案**：会

**说明**：
- 同一个帖子会生成相同的文件名
- 重复下载会覆盖旧文件
- 如果有新评论，文件会更新

## 最佳实践

### 首次使用

1. 安装 Playwright 浏览器
2. 运行全量下载：`/xz full`
3. 检查 output 目录中的文件
4. 验证内容是否正确

### 日常使用

1. 定期运行增量更新：`/xz`
2. 建议频率：每天 1-2 次
3. 如果长时间未更新，考虑运行全量下载

### 备份

定期备份以下内容：
- `output/` 目录（所有下载的内容）
- `.xz/sync_state.json`（同步状态）

### 性能优化

1. 使用增量更新代替全量下载
2. 避免在网络不稳定时运行
3. 不要频繁运行全量下载（可能触发限流）

## 技术细节

### 工作原理

1. **浏览器自动化**
   - 使用 Playwright 控制 Chromium 浏览器
   - 模拟真实用户操作

2. **网络拦截**
   - 监听浏览器的网络请求
   - 拦截知识星球 API 的响应
   - 提取 JSON 数据

3. **数据处理**
   - 解析 API 响应
   - 筛选目标作者的内容
   - 标准化数据结构

4. **内容下载**
   - 下载图片和附件到本地
   - 使用相对路径引用

5. **Markdown 生成**
   - 转换为格式化的 Markdown
   - 包含元数据和评论

### 为什么不直接调用 API？

知识星球的 API 需要认证，直接调用可能：
- 触发安全检测
- 需要复杂的认证流程
- 容易被封禁

浏览器自动化的优势：
- 使用已登录的会话
- 行为更接近真实用户
- 更稳定可靠

## 故障排除

### 查看日志

所有操作都会输出日志，包括：
- 时间戳
- 操作描述
- 错误信息

### 重置状态

如果遇到问题，可以删除状态文件重新开始：

```bash
rm .xz/sync_state.json
/xz full
```

### 清空输出

如果需要重新下载所有内容：

```bash
rm -rf output/*
rm .xz/sync_state.json
/xz full
```
