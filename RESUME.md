# 断点续传功能说明

## 概述

全量下载支持断点续传，下载过程中如果中断，下次运行会自动从上次的位置继续，不会重复下载。

## 特点

✅ **自动检测** - 无需手动操作，自动识别未完成的下载
✅ **智能恢复** - 从上次中断的时间点继续
✅ **进度保存** - 实时保存下载进度
✅ **多次恢复** - 支持多次中断和恢复
✅ **不重复下载** - 已下载的内容不会重复处理

## 使用方法

### 正常使用

```bash
# 首次运行全量下载
/xz full

# 如果中途中断了（网络断开、手动停止等）
# 再次运行相同命令即可继续
/xz full
```

### 进度信息

中断后再次运行会看到：

```
========================================
开始全量同步
========================================
⚠️  检测到未完成的下载
📍 将从上次中断位置继续（最早时间: 2026-01-15T10:00:00Z）
📊 已下载: 150 个帖子
```

下载过程中会显示进度保存信息：

```
💾 保存进度: 已下载到 2026-01-10T08:30:00Z, 共 200 个帖子
```

## 工作原理

### 1. 进度记录

下载过程中，工具会在 `.xz/sync_state.json` 中记录：

```json
{
  "full_sync_progress": {
    "in_progress": true,
    "oldest_time": "2026-01-15T10:00:00Z",
    "total_downloaded": 150
  }
}
```

- `in_progress`: 是否有进行中的下载
- `oldest_time`: 已下载到的最早时间戳
- `total_downloaded`: 已下载的帖子总数

### 2. 自动恢复

下次运行时：
1. 检查 `in_progress` 是否为 `true`
2. 如果是，从 `oldest_time` 继续往前（更早）下载
3. 已下载的内容（在 `oldest_time` 之后的）会被跳过

### 3. 完成标记

全部下载完成后：
```json
{
  "full_sync_progress": {
    "in_progress": false,
    "oldest_time": null,
    "total_downloaded": 0
  }
}
```

## 常见场景

### 场景 1: 网络不稳定

```bash
# 开始下载
/xz full
# ... 下载了 100 个帖子后网络断开 ...

# 网络恢复后，继续下载
/xz full
# ⚠️  检测到未完成的下载
# 📍 将从上次中断位置继续...
# ... 继续下载剩余内容 ...
```

### 场景 2: 手动停止

```bash
# 开始下载
/xz full
# ... 按 Ctrl+C 手动停止 ...

# 想继续时
/xz full
# 自动从上次位置继续
```

### 场景 3: 程序崩溃

```bash
# 开始下载
/xz full
# ... 程序意外崩溃 ...

# 重新运行
/xz full
# 自动恢复进度
```

### 场景 4: 分批下载

```bash
# 第一天下载一部分
/xz full
# ... 下载到某个时间点后手动停止 ...

# 第二天继续
/xz full
# 从第一天停止的地方继续
```

## 重新开始

如果不想继续，想重新开始全量下载：

```bash
# 删除状态文件
rm .xz/sync_state.json

# 重新开始
/xz full
```

或者只删除进度信息，保留已下载的帖子记录：

手动编辑 `.xz/sync_state.json`，将 `full_sync_progress` 修改为：

```json
{
  "full_sync_progress": {
    "in_progress": false,
    "oldest_time": null,
    "total_downloaded": 0
  }
}
```

## 注意事项

1. **增量更新不使用断点续传**
   - 断点续传只在全量下载时生效
   - 增量更新本身就很快，不需要断点续传

2. **浏览器关闭不影响进度**
   - 进度保存在文件中
   - 下次运行会打开新的浏览器继续

3. **已保存的帖子不会重新下载**
   - 已处理并保存为 Markdown 的帖子会被记录
   - 断点续传时这些帖子会被跳过

4. **进度保存频率**
   - 每次发现更早内容时自动保存
   - 每处理 5 个帖子保存一次
   - 确保进度不会丢失太多

## 故障排除

### 问题：一直显示"检测到未完成的下载"

**原因**：上次下载可能没有正常完成

**解决**：
1. 继续运行，让它下载完成
2. 或者删除状态文件重新开始

### 问题：断点续传后发现有重复的帖子

**原因**：可能是时间戳边界的帖子

**影响**：
- Markdown 文件会被覆盖更新
- 不会产生重复文件（文件名相同）
- 对最终结果无影响

### 问题：想查看当前进度

**方法**：
```bash
# 查看状态文件
cat .xz/sync_state.json | grep -A 5 "full_sync_progress"
```

## 技术实现

断点续传通过以下机制实现：

1. **实时进度保存** - 下载过程中定期保存进度到磁盘
2. **时间戳追踪** - 记录已下载到的最早时间点
3. **智能恢复** - 启动时检测并从中断点继续
4. **去重机制** - 确保不会重复处理相同内容
