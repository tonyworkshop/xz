亲自体验AI 编程能力进步的随想

现在的资本市场的叙事已经完全变成AI主导一切了，可怜的Crypto圈被打入了冷宫，所有所有的注意力和焦点都在AI这个赛道上，不体验一下最新的AI模型的范式转移是要被市场淘汰掉的。


我这两天高强度的Vibe Coding了一把，主要是用Google提供的工具Anti- Gravity，相比于Claude Code的命令行强迫症，让我更加舒服，工具没有绑定模型，除了Gemini3 之外，Claude 的模型也是可以用的。Codex 新版本也用了，这三个其实本质上是一个赛道的，封装的方式不一样而已。


做了两个项目，一个是期权降成本的管理工具，这个还没达到我满意的程度，另外一个是字幕自动翻译工具。Youtube 提供语言识别字幕（经常搞错）和自动翻译（旧版本Google Translate 机翻）质量经常很糟糕，前言不搭后语，和现在LLM非常博学的理解能力翻译地道语言完全不能相比，翻译质量天差地别。我当然理解以Youtube的海量视频都去烧Token做高质量翻译成本上吃不消的。但有些高质量的视频，我希望用高质量的LLM去翻译。


有了这个想法后，和Gemini3 商量了一下思路，就直接让开工了

成果开源在：https://github.com/xuzhe35/LLM-Subtitles


其实几个月前体验过 Vibe Coding ，那时候的感觉是有这个东西了方便多了，至少不用频繁的去翻API的文档说明了，但用着用着这玩意儿产生Bug的速度完全拖累掉了它提高效率的速度。有种和一个博学的但编码一直出纰漏的人结对编程的感觉。更痛苦的是，它经常会在自己的逻辑死穴里打转转，有种还不如老子自己从头写的感受。


那会儿只觉得这东西以后进步进步将来多半会有大用，前途是光明的，道路是曲折的，或许需要底层模型的原理有点范式突破啥的。


万万没想到，几个月而已，已经彻底变了。当我把软件的工作流程说了个大概之后，想要甩给模型看它出哪些问题，并且之前的经验是逻辑太复杂了它会遗忘很多的，所以我不敢说太多，可结果完全出乎了我的意料。Plan 模式给了我一个根据我的意思很详细的技术方案，我读后发现理解的复杂度远超之前的上限并且落实的步骤很具体，很可行，有种开完了研讨会的纪要的感觉。我试探性的点了 Proceed，想看一下第一个版本的代码质量，但心里做好了给模型擦屁股的准备。


万万没想道的是，第一个版本就能工具。我一行代码都没改，能工作的软件，就出现了。


心里只有强烈的震惊，涌现出的三个字是“变天了”，有种见鬼了的感觉，感觉如果我还在做程序员大概可以收拾纸箱子准备滚蛋了。


当然第一个版本是不可能让人完全满意的，随着自己使用的深入，问题还是层出不穷的出来，不是编码的正确性问题，而是之前很多细节没想道，以前做程序员的时候总是吐槽客户事儿逼，一会儿想一出是一出，现在屁股在提需求的这一侧了，完全就变成那副嘴脸了。


软件做出来后，拿Friedman 的采访视频做了一个测试，话题就是这个。真实的感受就是他们讨论的那个：工作流程的演变。



从第一刻感觉到好像技术人员没用了，这个软件根本不是我写的，到后来发现我依然是软件的创作者，而AI更像是一个超级智能的IDE，并非创造者主体。


要做什么，怎么做这个问题，依然要人来定义，而具体翻译成代码的工作，是AI大模型在做的。


从这个层面上来说，它的意义更像是那个机器码拥有了汇编，而C语言能让人看得懂的代码编程汇编那种跨越，只不过这次的跨越更大，从自然语言到了机器语言。


我是刚学编程的时候已经是高级程序语言的天下了，除了在编译原理的课上了解底层语言的逻辑之外，是从来不会写非高级程序语言的。并且热衷于抽象层次高的语法糖。


Vibe coding可以理解成一种终极的语法糖，甜度爆表。但我依然在定义工作流程，推演原理层面的问题，决策技术路线。只不过大量的技术执行工作都被AI完成掉了，我感受到的是那种出现了全新的框架后的能力提升，而非我不在写软件了。


本来以为开着Anti Gravity 几次 Prompt 就搞定了，但其实不是的，这个软件对我来说依然是很强的IDE的感觉，对着它莫名其妙就过了一天了，搞到深更半夜了，像极了以前高强度搞开发的时候。


我依然是软件的作者，这个是这几天高强度 Vibe Coding的感受。



好多次效果都是不好的，然后找问题，尝试方案，最后声音和内容完全能匹配上，字幕显示也很舒服。因为大量的底层技术实现不用手动了，开发的乐趣其实被保留了。


最大的一个坎是Whisper模型对小语种的识别经常“偷懒”，于是我突然想到Google原来那个完全几乎HMM的传统古董级模型不会有LLM based模型那样的语义癖，绝对不会偷摸不翻。这个对于我要翻译泰文开示来说是一个纠缠了两天的BUG，想到这个技术路线后让LLM去做了，完全没有读一个新API文档的痛苦，博学的Gemini3 完全自己都搞定了，我只要给API Key 就完事儿了。


最后问题如预期的解决，这个问题是我解决的还是AI解决的？我想是我，AI编码实现了。还是人在编程。


但完全是我吗？也不是，可以说是一种“共创”，但人并没有被彻底取代掉。那种惊呼已经过了门槛，人类没用的感受在体验的前几个小时后就消失了。


让我印象最深的是因为这个应用是完全让AI去用AI的，让Gemini3 写了一个 GPT API 的程序，我发现了AI 可以Prompt AI，这就很有意思了。



这个完全是Gemini3 自己想出来的，关于Gemini3 如何使用GPT的有趣场景就这样水灵灵的呈现在我面前。


这个递归有点细思极恐的，那根据AI模型对AI模型的效果的反馈持续迭代的话，能出现什么呢？


Codex 据说已经是 Codex写的了，Claude Code 基本是 Claude Code 写的了，未来会发生什么？


不知道


但我知道一个全新的人机交互的时代已经来了，跨度比以往任何一次都大，而自动化这件事就是技术革新的一个再明确不过的信号了。


AI 技术革命里有没有估值泡沫？我想肯定是有的，但未来技术带来的真金白银的回报将会是现在这个庞大泡沫的很多很多倍恐怕也不是什么难以想象的事情，我甚至不认为是个肥尾小概率事件，恐怕是个大概率事件。


现在看似天文数字的海量投入资本支出，和这个体量的技术进步相比，都是小意思。


继续做多AI技术未来的右侧尾部，是这次体验给我强烈的信号。
