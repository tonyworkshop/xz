元宝智能体使用界面




“预期波动计算器”是一个便捷的期权辅助小工具，旨在帮助期权投资者快速估算在给定一段时间内，基于当前标的资产价格和隐含波动率 (IV)，其价格可能波动的范围。

这个工具的核心是一个 Python 脚本，它能解析你输入的参数，并结合市场规则和日期计算，给出一个预期的价格高点和低点。理解这个预期范围，对于选择合适的看涨期权 (Call) 和看跌期权 (Put) 的行权价非常有帮助。

它有两个主要的使用方式：

  1. 1. Alfred Workflow：专为 macOS 用户设计，可以深度集成到 Alfred 中，通过快捷键和关键词快速调用。
  2. 2. 腾讯元宝智能体 (Agent)：作为一个智能体应用，它可以在支持腾讯元宝智能体的平台上运行，通过对话或命令方式调用。


计算原理概述


这个计算器的核心原理基于金融学中一个常用的概念：隐含波动率 (IV)。简单来说：

  1. 1. 隐含波动率 (IV)：这是市场对未来价格波动剧烈程度的预期。通常以年化百分比表示。IV 越高，意味着市场预期价格波动越大。脚本允许你输入看涨期权的 IV 和看跌期权的 IV，如果只提供一个，则认为两者相同，并取其平均值作为计算基准。
  2. 2. 时间因素：波动范围的估算与时间长短有关。距离到期日越近，预期的绝对波动范围通常越小。脚本通过公式 sqrt(剩余天数 / 365) 来调整年化的 IV，将其转换为到期日对应的波动率因子。
  3. 3. 预期波动金额：通过 当前价格 * (平均IV / 100) * (时间调整因子) 计算得出。
  4. 4. 预期价格区间：
  5. • 预期高点 = 当前价格 + 预期波动金额
  6. • 预期低点 = 当前价格 - 预期波动金额

脚本还会处理不同市场的交易日规则（目前主要针对美股市场做了简单的盘前/盘后日期调整），并能灵活解析多种日期输入格式。

重要提示：这只是一个基于简化模型的估算，实际市场价格会受到多种复杂因素的影响，包括但不限于突发新闻、宏观经济数据、市场情绪等。因此，计算结果仅供参考，不构成任何投资建议或对行权价选择的绝对指导。


如何使用

1. 作为 Alfred Workflow


假设你已经按照之前的说明，将 Python 脚本配置为一个 Alfred Workflow，并设置了触发关键词（例如 emc）。

使用步骤：

  1. 1. 激活 Alfred：通常是按下 Option + Space (macOS 默认快捷键)。
  2. 2. 输入命令和参数：在 Alfred 输入框中，输入你的关键词，后跟你想要计算的参数，参数之间用空格或逗号隔开。
  3. • 格式为：关键词 价格 [看涨IV] [看跌IV 或 单个IV] 日期 [市场类型]
  4. • 价格：必填，当前的资产价格。
  5. • 看涨IV / 看跌IV / 单个IV：必填，隐含波动率，输入百分比数字即可（例如，输入 25 代表 25%）。
  6. • 如果提供两个 IV 值，则它们分别代表看涨和看跌 IV。
  7. • 如果只提供一个 IV 值，则脚本会认为看涨和看跌 IV 相同。
  8. • 日期：必填，期权或预期的到期日。支持以下格式：
  9. • YYYYMMDD (例如：20241231)
  10. • MMDD (自动使用当年，例如：1231)
  11. • DD (自动使用当年当月，例如：15 表示当月的15号)
  12. • 市场类型：可选，默认为 'us' (美国市场)。支持：
  13. • us: 美国市场 (日期计算会考虑美股交易时间特点做简单调整，不写默认值是这个)
  14. • hk: 香港市场
  15. • a: A股市场
  16. 3. 查看结果：输入完毕后，Alfred 的 Text View 窗口会弹出，以 Markdown 格式显示详细的计算结果，包括：
  17. • 你输入的原始参数
  18. • 当前价格、到期日、市场类型、剩余天数
  19. • 预期波动百分比、预期波动金额
  20. • 预期的高点和低点

Alfred 输入示例：

  1. • emc 100 25 20241231 us
  2. • emc 150.5 30.2 31.5 1231
  3. • emc 2000 18 15 a
  4. • emc 55,22,0830,hk


2. 作为腾讯元宝智能体 (Agent)


该脚本也被封装为一个腾讯元宝智能体。其核心计算逻辑与 Alfred Workflow 版本相同。

使用方式：

具体的使用方式取决于腾讯元宝智能体平台的交互界面。通常，你会通过类似聊天或命令输入的方式与智能体互动。

  1. • 输入格式：与 Alfred Workflow 类似，你需要提供价格、IV（一个或两个）、日期、以及可选的市场类型。这些参数同样可以用空格或逗号分隔。
  2. • 例如，你可能会对智能体说或输入： 481.2 36 34 27 hk (其中 27 是日期，hk 是市场)。
  3. • 输出结果：智能体会返回与 Alfred 版本类似的计算结果，包括到期天数、预期涨跌幅、预期的上涨价格和下跌价格。输出的格式会根据智能体平台的显示能力而定，例如你提供的截图中显示的是结构化的文本信息。

错误处理：

无论是 Alfred Workflow 还是腾讯元宝智能体，如果你的输入格式不正确，或者参数不符合逻辑（例如到期日在今天之前），脚本会尝试给出相应的错误提示或无法计算的反馈。


适用范围与场景


这个工具主要适用于：

  1. • 辅助选择期权行权价：这是该工具的核心应用场景。通过估算标的资产在期权到期前的可能波动范围（即预期高点和低点），投资者可以更有依据地选择：
  2. • 看涨期权 (Call)：可能会考虑选择预期高点附近或略高于当前价的行权价。
  3. • 看跌期权 (Put)：可能会考虑选择预期低点附近或略低于当前价的行权价。
  4. • 对于期权卖方，这个范围可以帮助判断哪些行权价的期权金可能比较“安全”。
  5. • 快速估算短期波动：对于持有股票、期权或参与其他金融产品交易的个人，可以快速了解在特定 IV 和时间周期下，价格可能变动的范围。
  6. • 期权策略制定参考：在制定期权策略时（如卖出跨式/宽跨式期权、构建牛市/熊市价差等），可以用来估算一个相对合理的行权价组合范围。
  7. • 风险意识提醒：通过了解潜在的波动范围，帮助用户对期权投资的潜在风险和收益有一个大致的预期。
  8. • 学习和研究：对于学习金融衍生品和波动率概念的用户，这是一个直观的辅助工具，帮助理解 IV 和时间对价格预期的影响。

不适用或需谨慎使用的场景：

  1. • 精确预测与唯一决策依据：再次强调，这不是一个精确的价格预测工具，其结果不应作为选择行权价的唯一标准。
  2. • 重大新闻或事件期间：在有重大财报、政策发布等可预期会引起市场剧烈波动的事件前后，单纯依赖历史或当前 IV 可能不足以反映真实风险。
  3. • 流动性差的产品：对于流动性较差的期权合约或标的资产，其 IV 可能不准确或容易失真，计算结果的参考价值会降低。
  4. • 长期期权：时间越长，不确定性因素越多，该模型的估算准确性会下降，对于远期期权的行权价选择参考意义有限。

# -*- coding: utf-8 -*-
import sys
import json
import datetime
import math
import re

# ------------------------------------------------------------------------------
# 核心计算函数: calculate_expected_move_dict_from_string_no_pytz
# ------------------------------------------------------------------------------
def calculate_expected_move_dict_from_string_no_pytz(input_string: str) -> dict:
"""
解析3, 4, 或 5部分字符串,计算预期波动,返回字典(无pytz)。

输入格式 (空格/逗号分隔):
- 5 部分: 价格 看涨IV 看跌IV 日期 市场类型 (IV可不同)
- 4 部分(A): 价格 看涨IV 看跌IV 日期 (IV可不同, 市场默认'us')
- 4 部分(B): 价格 相同IV 日期 市场类型 (IV相同)
- 3 部分: 价格 相同IV 日期 (IV相同, 市场默认'us')

- 日期格式: 'yyyymmdd', 'mmdd'(补当年), 'dd'(补当年当月)。
- 市场类型: 'us', 'hk', 'a'。可选, 默认'us'。影响日期比较基准。
- 天数计算: 基于日期差 (到期日 - 市场当前日期)。结果 > 0。
- US市场日期判断: 使用简单UTC小时启发式规则。
- IV: 若只提供一个IV值，则假定看涨=看跌=该值。要求非负。

参数 (Args):
input_string (str): 含3, 4, 或 5 部分的字符串。

返回 (Returns):
dict: {'current_price', 'days_to_expiry', 'expected_move_percent',
'expected_high', 'expected_low', 'market_type', 'expiry_date_str'}

异常 (Raises):
ValueError: 输入格式/部分数量(非3,4,5)/值/类型错误, 市场类型无效,
日期格式/值无效, 到期日不晚于市场当前日期。
RuntimeError: 其他计算错误。
"""
# --- 1. 解析输入字符串 ---
parts = re.split(r'[,\s]+', input_string.strip())
parts = [p for p in parts if p] # 移除空字符串
num_parts = len(parts)

market_type = 'us' # 默认市场类型
valid_markets = {'us', 'hk', 'a'}
price_str, call_iv_str, put_iv_str, expiry_date_str_raw = "", "", "", ""

if num_parts == 5:
price_str, call_iv_str, put_iv_str, expiry_date_str_raw, market_type_raw = parts
market_type_input = market_type_raw.strip().lower()
if market_type_input not in valid_markets:
raise ValueError(f"无效的市场类型 '{market_type_raw.strip()}'。有效值为 'us', 'hk', 'a'。")
market_type = market_type_input
elif num_parts == 4:
potential_market = parts[3].strip().lower()
if potential_market in valid_markets: # 格式: 价格 相同IV 日期 市场
price_str = parts[0]
call_iv_str = parts[1]
put_iv_str = parts[1]
expiry_date_str_raw = parts[2]
market_type = potential_market
else: # 格式: 价格 看涨IV 看跌IV 日期 (市场默认 'us')
price_str = parts[0]
call_iv_str = parts[1]
put_iv_str = parts[2]
expiry_date_str_raw = parts[3]
elif num_parts == 3: # 格式: 价格 相同IV 日期 (市场默认 'us')
price_str = parts[0]
call_iv_str = parts[1]
put_iv_str = parts[1]
expiry_date_str_raw = parts[2]
else:
raise ValueError(f"输入字符串必须包含3, 4, 或 5个由空格或逗号分隔的部分，实际得到 {num_parts} 部分。")
expiry_date_input_str = expiry_date_str_raw.strip()

# --- 2. 转换并验证价格和 IVs ---
try:
current_price = float(price_str.strip())
if current_price <= 0: raise ValueError(f"价格 '{current_price}' 必须为正数。")
except ValueError as e:
if "convert" in str(e): raise ValueError(f"价格部分 '{price_str.strip()}' 无法转换为数字。") from e
else: raise e
try:
atm_call_iv = float(call_iv_str.strip())
if atm_call_iv < 0: raise ValueError(f"看涨(或相同)IV '{atm_call_iv}' 必须非负。")
except ValueError as e:
if "convert" in str(e): raise ValueError(f"看涨(或相同)IV部分 '{call_iv_str.strip()}' 无法转换为数字。") from e
else: raise e
try:
atm_put_iv = float(put_iv_str.strip())
if atm_put_iv < 0: raise ValueError(f"看跌(或相同)IV '{atm_put_iv}' 必须非负。")
except ValueError as e:
if "convert" in str(e): raise ValueError(f"看跌(或相同)IV部分 '{put_iv_str.strip()}' 无法转换为数字。") from e
else: raise e

# --- 3. 确定日期并计算天数差异 ---
try:
now_local = datetime.datetime.now()
today_user = now_local.date()
current_year = today_user.year
current_month = today_user.month
now_utc = datetime.datetime.now(datetime.timezone.utc)
utc_hour_now = now_utc.hour

date_len = len(expiry_date_input_str)
if date_len == 8: full_date_str = expiry_date_input_str; fmt = '%Y%m%d'
elif date_len == 4: full_date_str = f"{current_year}{expiry_date_input_str}"; fmt = '%Y%m%d'
elif date_len == 2: full_date_str = f"{current_year}{current_month:02d}{expiry_date_input_str}"; fmt = '%Y%m%d'
else: raise ValueError(f"到期日 '{expiry_date_input_str}' 长度无效。请使用 'YYYYMMDD', 'MMDD', 或 'DD' 格式。")

try:
expiry_dt_naive = datetime.datetime.strptime(full_date_str, fmt)
expiry_date = expiry_dt_naive.date()
except ValueError as e_fmt:
raise ValueError(f"根据输入 '{expiry_date_input_str}' 解析出的日期字符串 '{full_date_str}' 无效。") from e_fmt

if market_type == 'us':
today_market = today_user - datetime.timedelta(days=1) if utc_hour_now < 5 else today_user
else:
today_market = today_user

date_diff = expiry_date - today_market
if date_diff.days <= 0:
raise ValueError(f"目标到期日 '{expiry_date}' (来自输入 '{expiry_date_input_str}') 必须严格晚于市场当前日期 '{today_market}' (市场: {market_type.upper()})。")
days_to_expiry_int = date_diff.days
except ValueError as ve: raise ve
except Exception as e: raise RuntimeError(f"处理日期或计算天数时发生意外错误: {e}")

# --- 4. 计算平均 IV ---
iv_call_decimal = atm_call_iv / 100.0
iv_put_decimal = atm_put_iv / 100.0
iv = (iv_call_decimal + iv_put_decimal) / 2.0
# --- 5. 执行波动计算 ---
try:
time_factor = math.sqrt(days_to_expiry_int / 365.0) if days_to_expiry_int > 0 else 0
expected_move_money = current_price * iv * time_factor
expected_move_percent = (expected_move_money / current_price) * 100 if current_price != 0 else 0
expected_high = current_price + expected_move_money
expected_low = current_price - expected_move_money
except Exception as e:
raise RuntimeError(f"计算预期波动时发生错误: {e}")

# --- 6. 准备输出字典 ---
result_dict = {
'current_price': current_price,
'days_to_expiry': days_to_expiry_int,
'expected_move_percent': round(expected_move_percent, 2),
'expected_high': round(expected_high, 2),
'expected_low': round(expected_low, 2),
'market_type': market_type,
'expiry_date_str': expiry_date.strftime('%Y-%m-%d')
}
return result_dict

这个计算器的核心原理基于金融学中一个常用的概念：隐含波动率 (IV)。简单来说：

  1. 1. 隐含波动率 (IV)：这是市场对未来价格波动剧烈程度的预期。通常以年化百分比表示。IV 越高，意味着市场预期价格波动越大。脚本允许你输入看涨期权的 IV 和看跌期权的 IV，如果只提供一个，则认为两者相同，并取其平均值作为计算基准。
  2. 2. 时间因素：波动范围的估算与时间长短有关。距离到期日越近，预期的绝对波动范围通常越小。脚本通过公式 sqrt(剩余天数 / 365) 来调整年化的 IV，将其转换为到期日对应的波动率因子。
  3. 3. 预期波动金额：通过 当前价格 * (平均IV / 100) * (时间调整因子) 计算得出。
  4. 4. 预期价格区间：
  5. • 预期高点 = 当前价格 + 预期波动金额
  6. • 预期低点 = 当前价格 - 预期波动金额

脚本还会处理不同市场的交易日规则（目前主要针对美股市场做了简单的盘前/盘后日期调整），并能灵活解析多种日期输入格式。

重要提示：这只是一个基于简化模型的估算，实际市场价格会受到多种复杂因素的影响，包括但不限于突发新闻、宏观经济数据、市场情绪等。因此，计算结果仅供参考，不构成任何投资建议或对行权价选择的绝对指导。

  1. • 辅助选择期权行权价：这是该工具的核心应用场景。通过估算标的资产在期权到期前的可能波动范围（即预期高点和低点），投资者可以更有依据地选择：
  2. • 看涨期权 (Call)：可能会考虑选择预期高点附近或略高于当前价的行权价。
  3. • 看跌期权 (Put)：可能会考虑选择预期低点附近或略低于当前价的行权价。
  4. • 对于期权卖方，这个范围可以帮助判断哪些行权价的期权金可能比较“安全”。
  5. • 快速估算短期波动：对于持有股票、期权或参与其他金融产品交易的个人，可以快速了解在特定 IV 和时间周期下，价格可能变动的范围。
  6. • 期权策略制定参考：在制定期权策略时（如卖出跨式/宽跨式期权、构建牛市/熊市价差等），可以用来估算一个相对合理的行权价组合范围。
  7. • 风险意识提醒：通过了解潜在的波动范围，帮助用户对期权投资的潜在风险和收益有一个大致的预期。

  4. • 长期期权：时间越长，不确定性因素越多，该模型的估算准确性会下降，对于远期期权的行权价选择参考意义有限。

# -*- coding: utf-8 -*-
import sys
import json
import datetime

# ------------------------------------------------------------------------------
# 核心计算函数: calculate_expected_move_dict_from_string_no_pytz
# ------------------------------------------------------------------------------
def calculate_expected_move_dict_from_string_no_pytz(input_string: str) -> dict:
"""
解析3, 4, 或 5部分字符串,计算预期波动,返回字典(无pytz)。

输入格式 (空格/逗号分隔):
- 5 部分: 价格 看涨IV 看跌IV 日期 市场类型 (IV可不同)
- 4 部分(A): 价格 看涨IV 看跌IV 日期 (IV可不同, 市场默认'us')
- 4 部分(B): 价格 相同IV 日期 市场类型 (IV相同)
- 3 部分: 价格 相同IV 日期 (IV相同, 市场默认'us')

- 日期格式: 'yyyymmdd', 'mmdd'(补当年), 'dd'(补当年当月)。
- 市场类型: 'us', 'hk', 'a'。可选, 默认'us'。影响日期比较基准。
- 天数计算: 基于日期差 (到期日 - 市场当前日期)。结果 > 0。
- US市场日期判断: 使用简单UTC小时启发式规则。
- IV: 若只提供一个IV值，则假定看涨=看跌=该值。要求非负。

参数 (Args):
input_string (str): 含3, 4, 或 5 部分的字符串。

返回 (Returns):
dict: {'current_price', 'days_to_expiry', 'expected_move_percent',
'expected_high', 'expected_low', 'market_type', 'expiry_date_str'}

异常 (Raises):
ValueError: 输入格式/部分数量(非3,4,5)/值/类型错误, 市场类型无效,
日期格式/值无效, 到期日不晚于市场当前日期。
RuntimeError: 其他计算错误。
"""
# --- 1. 解析输入字符串 ---
parts = re.split(r'[,\s]+', input_string.strip())
parts = [p for p in parts if p] # 移除空字符串
num_parts = len(parts)

market_type = 'us' # 默认市场类型
valid_markets = {'us', 'hk', 'a'}
price_str, call_iv_str, put_iv_str, expiry_date_str_raw = "", "", "", ""

if num_parts == 5:
price_str, call_iv_str, put_iv_str, expiry_date_str_raw, market_type_raw = parts
market_type_input = market_type_raw.strip().lower()
if market_type_input not in valid_markets:
raise ValueError(f"无效的市场类型 '{market_type_raw.strip()}'。有效值为 'us', 'hk', 'a'。")
market_type = market_type_input
elif num_parts == 4:
potential_market = parts[3].strip().lower()
if potential_market in valid_markets: # 格式: 价格 相同IV 日期 市场
price_str = parts[0]
call_iv_str = parts[1]
put_iv_str = parts[1]
expiry_date_str_raw = parts[2]
market_type = potential_market
else: # 格式: 价格 看涨IV 看跌IV 日期 (市场默认 'us')
price_str = parts[0]
call_iv_str = parts[1]
put_iv_str = parts[2]
expiry_date_str_raw = parts[3]
elif num_parts == 3: # 格式: 价格 相同IV 日期 (市场默认 'us')
price_str = parts[0]
call_iv_str = parts[1]
put_iv_str = parts[1]
expiry_date_str_raw = parts[2]
else:
raise ValueError(f"输入字符串必须包含3, 4, 或 5个由空格或逗号分隔的部分，实际得到 {num_parts} 部分。")
expiry_date_input_str = expiry_date_str_raw.strip()

# --- 2. 转换并验证价格和 IVs ---
try:
current_price = float(price_str.strip())
if current_price <= 0: raise ValueError(f"价格 '{current_price}' 必须为正数。")
except ValueError as e:
if "convert" in str(e): raise ValueError(f"价格部分 '{price_str.strip()}' 无法转换为数字。") from e
else: raise e
try:
atm_call_iv = float(call_iv_str.strip())
if atm_call_iv < 0: raise ValueError(f"看涨(或相同)IV '{atm_call_iv}' 必须非负。")
except ValueError as e:
if "convert" in str(e): raise ValueError(f"看涨(或相同)IV部分 '{call_iv_str.strip()}' 无法转换为数字。") from e
else: raise e
try:
atm_put_iv = float(put_iv_str.strip())
if atm_put_iv < 0: raise ValueError(f"看跌(或相同)IV '{atm_put_iv}' 必须非负。")
except ValueError as e:
if "convert" in str(e): raise ValueError(f"看跌(或相同)IV部分 '{put_iv_str.strip()}' 无法转换为数字。") from e
else: raise e

# --- 3. 确定日期并计算天数差异 ---
try:
now_local = datetime.datetime.now()
today_user = now_local.date()
current_year = today_user.year
current_month = today_user.month
now_utc = datetime.datetime.now(datetime.timezone.utc)
utc_hour_now = now_utc.hour

date_len = len(expiry_date_input_str)
if date_len == 8: full_date_str = expiry_date_input_str; fmt = '%Y%m%d'
elif date_len == 4: full_date_str = f"{current_year}{expiry_date_input_str}"; fmt = '%Y%m%d'
elif date_len == 2: full_date_str = f"{current_year}{current_month:02d}{expiry_date_input_str}"; fmt = '%Y%m%d'
else: raise ValueError(f"到期日 '{expiry_date_input_str}' 长度无效。请使用 'YYYYMMDD', 'MMDD', 或 'DD' 格式。")

try:
expiry_dt_naive = datetime.datetime.strptime(full_date_str, fmt)
expiry_date = expiry_dt_naive.date()
except ValueError as e_fmt:
raise ValueError(f"根据输入 '{expiry_date_input_str}' 解析出的日期字符串 '{full_date_str}' 无效。") from e_fmt

if market_type == 'us':
today_market = today_user - datetime.timedelta(days=1) if utc_hour_now < 5 else today_user
else:
today_market = today_user

date_diff = expiry_date - today_market
if date_diff.days <= 0:
raise ValueError(f"目标到期日 '{expiry_date}' (来自输入 '{expiry_date_input_str}') 必须严格晚于市场当前日期 '{today_market}' (市场: {market_type.upper()})。")
days_to_expiry_int = date_diff.days
except ValueError as ve: raise ve
except Exception as e: raise RuntimeError(f"处理日期或计算天数时发生意外错误: {e}")

# --- 4. 计算平均 IV ---
iv_call_decimal = atm_call_iv / 100.0
iv_put_decimal = atm_put_iv / 100.0
iv = (iv_call_decimal + iv_put_decimal) / 2.0
# --- 5. 执行波动计算 ---
try:
time_factor = math.sqrt(days_to_expiry_int / 365.0) if days_to_expiry_int > 0 else 0
expected_move_money = current_price * iv * time_factor
expected_move_percent = (expected_move_money / current_price) * 100 if current_price != 0 else 0
expected_high = current_price + expected_move_money
expected_low = current_price - expected_move_money
except Exception as e:
raise RuntimeError(f"计算预期波动时发生错误: {e}")

# --- 6. 准备输出字典 ---
result_dict = {
'current_price': current_price,
'days_to_expiry': days_to_expiry_int,
'expected_move_percent': round(expected_move_percent, 2),
'expected_high': round(expected_high, 2),
'expected_low': round(expected_low, 2),
'market_type': market_type,
'expiry_date_str': expiry_date.strftime('%Y-%m-%d')
}
return result_dict
