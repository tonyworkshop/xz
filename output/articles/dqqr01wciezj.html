据我所知，我一无所知

最近中信出版社送了我一套塔勒布丛书的完整版，大概是因为之前比较难度的《肥尾分布》的书本来销量一般，我写了一篇简介之后销量很有起色。然后又给我邮寄了一套芒格的语录，芒格挂了，现在给我送了塔勒布全集，然后塔勒布的信众被提名美国财长了。这是流量追着我，热点怼我脸，容易写爆款，但我偏不！



先说一下购买建议：如果你大学里关于概率论统计学不是很认真的话，《肥尾效应》确实不太合适读。我第一次读的时候，也把大学时期的教材翻开来复习了好久，然后又在油管里找了几个印度老哥补习了下。如果你觉得自己脱离大学数学太远了，买了多半啃不下去。


《黑天鹅》《随机漫步的傻瓜》和《反脆弱》是比较推荐阅读的，无论你是不是搞衍生品投资交易的，这些理念都是可以了解一下。《反脆弱》是与交易策略构造者而言是必读的，其理念是前两本书的一个加强，主旨是完全一致的。《非对称风险》是关注目前整个经济制度的批判性看法，如果你对为什么当今世界经济在搞什么把戏，可以阅读一下，不读的话也不影响其他推论的影响。


至于那本《智慧箴言》，个人觉得还不如直接去看他的推，基本就是一个日常毒舌的集合而已，当消遣就行。


当然如果你和我一样，特别喜欢这个毒舌老头，买一套当收藏也可以的。


要理解塔勒布系列丛书的思想，还要回到最初成名的《黑天鹅》的关于随机性和不可预测的论述。《黑天鹅》能变成畅销书，是因为说的特别散文化，看着很好懂，但其实后面的道理很深。


我本来写文的特色也是通俗好懂，今天性情大变一下，搞点形式化的东西。我很确定大部分人看着会头晕，一点不好懂，但我的地盘我做主，今天就放肆发泄一回。


我保证下面的东西看不看对你的投资交易实践毫无影响，纯纯个人发癫，如果有人看着还觉得欢喜，也是快事。



黑天鹅一词来自于英语俚语，意思是不可能的事情，类似于中国人讲的「龟毛兔角」，也就是不存在的东西。因为英国人觉得所有的天鹅理所当然都应该是白色的。直到在澳大利亚发现了黑色的天鹅，这个词的含义瞬间变了，变成了「以前没有经历过的意外」。


这个有意思的小转变背后的哲学化反思通常被叫做「休谟问题」。意思是如果我们从过去的历史经验里提取总结出所谓的规律，那么好，问题是人类的知识穷尽了所有经验了吗？


如果人们从一个经验集合里总结出来了某种自以为是「规律」的东西，那么未来出现了一个之前从未出现过的新现象，并且它不符合总结出来的规律，且这种冒出「未有之大变局」的事情不知道什么时候会降临。那总结规律这件事情本身成立吗？


见过一万只天鹅是白色的也不能归纳成所有天鹅都是白色的，而有一只黑天鹅就能证伪天鹅都是白色的这个命题。


且最最要命的是：你真不知道啥时候会突然飞出一只黑天鹅来。


如果归纳总结规律这件事情本身不可靠，那么因果推断的所有手法是否值得怀疑呢？太阳暴晒过的石头变热了，我们能认为太阳暴晒是石头变热的「因」吗？事实上我们只是看到了两个先后顺序的现象而已，如果有一天出现了暴晒下不变热的石头怎么整？那所有的归纳总结都未必可靠的情况下，甚至整个因果体系是否可靠呢？


不要以为这个就是纯哲学的头脑体操，你所有瞪眼看出的所谓的「K线规律」（别名技术分析）和所有的「基本面因子」全部在归纳法的范畴里。无论你多量化，用多少统计学工具，都在这个范畴里逃不掉。但凡是在过往的历史数据里，无论是量价关系还是基本面信息乃至宏观经济分析，没有不是历史经验归纳法的。而归纳法，本身不可靠。


这个哲学流派有点接近古希腊的「怀疑主义哲学」，值得注意的是怀疑主义并不等价于全然的不可知论。怀疑主义的涵义是：不一定可靠，而彻底的不可知论是严格的你无法知道，这两者是有本质区别的。


远到古希腊的皮浪，近到塔勒布这样的怀疑论者都被人误以为是啥结论都不能下的怪人，毫无意义的偏执狂。《黑天鹅》里塔勒布甚至说有人问他你敢不敢过马路，这都是对怀疑论者的误解，因为你不知道下一秒是否突然有失控的大卡车嘛~~


怀疑论的意思是这些都没有那么值得完全确信，而不是所有的事情都是假的，值得怀疑本身就是一个信息，而非不可知；且已证伪的事情就是一个确凿的信息（很重要），界定上划分的话它是可知的。也就是承认休谟问题的存在并不一定滑向不可知与拔无因果，它们是泾渭分明的两个世界观。


休谟问题只是否认了一般性规律归纳法的可靠性，并不否认所有理性的可靠性。事实上纯的演绎方法的可靠性是可以让我们获得很多认知的，只要反过来就豁然开朗了，接下来我会大量的使用反证法来绕开休谟问题，或者说不用归纳法也是可以获得知识。怀疑论不等于不可知，怀疑论不等于无知识。


人们对于休谟问题的抗拒主要来自于人类自然科学上获得的成就，它确实让工业革命后人类的生存水平极大提高了，可以说工业革命是人类历史上唯一重要的事件。而科学革命就是建立在归纳总结上的，也就是休谟认为不可靠的办法上。科学恰恰就是靠「我见到的天鹅都是白色的，我们认为天鹅是白色的」基础之上的，它切切实实应用了。


嘿嘿~~是不是休谟老哥自己遇上自己的黑天鹅时刻了呢？


关于这个诘难，我见过的最有力的反驳来自于波普尔。他的原话我背不出，大致意思是：如果整个科学都建立在归纳法上，而他违背休谟问题提出的归纳的根本缺陷，我们同样不能先建立科学方法是对的假设，因为那也是基于我们现在的归纳上的，而不应该排除整个科学方法都是有问题的可能。这个纯纯的理性主义，让我觉得波普尔当得起汉传佛典里关于“狮子吼”的比喻，堪称振聋发聩，第一次读之时的心灵震撼记忆犹新。


不要因为自己想要急于知道些什么而迫不及待扑向归纳主义，《黑天鹅》里大量描述了急于知道原因造成的荒诞现象，这些都是正常人类的弱点，有谁希望站在无知的荒原上惶恐不安呢？让我知道点什么，确信点什么吧！啥都不确定是否太让人手足无措了？！处于不安，人类的大脑会大量的使用技能「脑补」，这是极难克服的。


不急不急，咱们来通过飞出黑天鹅的例子来体会如何反向获得确定性的知识，并且用这个知识再反过来推导休谟问题本身为什么蕴含着不可约性。


选的第一个例子是证明「图灵停机问题」的便捷方法。


图灵停机问题是问能否通过一个确定的算法，来判断一段程序是否会结束，也就是是否会陷入死循环。


直觉上，我们写代码的时候有时能瞪眼看出或者一步步Debug推演出一些程序会陷入死循环。这和用一种确定性的算法输入一段程序返回一个Bool返回值是否会陷入死循环是不一样的。也就是我们要写出一个方法，它的输入是一个函数，并且给定运行的参数，具体的算法要返回明确的True或False来判断这个传进去的函数是否能结束。


难道要穷举所有的算法来验证这个问题吗？这是不可能的。我们也不能靠列举出一百万个程序最后都停下来这个事情来归纳出图灵停机问题的答案是所有代码最后都会停下来（归纳的不可靠性）。


但我们只需要通过一个反例就能获取确凿无疑的知识。


假设我们现在确实有一个算法能实现图灵停机判定，这个Halts算法能返回明确的Ture或False来判断一段程序program是否会在对应的输入input的条件下停机。我们假装我们已经有这个算法了。

如果Halts方法判断这个输入的程序program是能停机的，也就是program不会陷入死循环，那么立马就执行一个死循环。如果Halts方法判断一个程序是不会停机的，就立马直接退出函数，并返回“已停机”。

如果判断它是不可停机的，那么就立刻停了它～，那么这个Halts方法判定的结果又不对了。

这是不是听起来像一个坏坏的恶作剧？嗯，它就是！


用这个自相矛盾的例子就能确凿的证明：图灵停机问题的判定函数是不可能存在的，这个Halts算法是绝对不可能写出来的。


我们提出一个假设，并且把这个假设的黑天鹅给飞出来，因为只要有一个黑天鹅的例子就能证伪一个命题，于是我们就能证明这个假设是错的，这样就证明了这个假设的反命题是真的了。这个是我们所有人在数学课堂上都学过的反证法，并不玄奥。上面这个恶作剧式的程序就是图灵停机问题的反证法，非常简洁易懂。


简单归简单，甚至有点玩笑的味道，但想想它的意义：不依赖归纳的真命题出炉了。


构造一个巧妙的悖论是困难的，自指代问题构造起悖论来简直批量可复现。下面我再来用类似的构造自指悖论的反证方法论，得出一个新的知识：柯尔莫罗夫复杂度（Kolmogorov Complexity）不可得。



（事实上这种自指代反证法思路能快捷证明出很多非常困难的问题，包括大名鼎鼎的哥德尔完备性定律和不完备性定律，有兴趣的小伙伴们可以参考视频：https://www.bilibili.com/video/BV1o7421K7pY/?share_source=copy_web&vd_source=ed20ca80cf65eaa0b1224329aaebbb71）



柯尔莫罗夫复杂度这个概念名字里带着复杂两个字，但这个的东西并不复杂，它指能压缩一段字符串的最短长度。比如我给你一串字符串“a@%!fdg8te"，它毫无规律，那么它的最短压缩长度也就是柯尔莫罗夫复杂度约等于它本身。而如果是一个字符串是“abcdefghigklmnopqrstuvwxyz"，那么我可以描述成英语字母表小写，它就被我用描述压缩了。我要还原的话，通过这句话（可以看作一个程序）就能还原。如果一个字符串是个字母表重复10遍，那么这个字符串长达260个字符，我可以通过一个描述“重复10遍英语字母表小写”去压缩这段话。



构造这么一个程序计算K(S)是不可能的，如果存在这么一个函数，那么我们写一个很奇葩的程序Q，要求


“必须大于100行代码才能生成的字符串”


但事实上，就用了一行描述就已经描述了它，明显小于100行，那么这个玩意儿Q就是不存在的。它天生就是悖论的，它自己的定义就让它陷入自我矛盾了。


而如果真的有一个算法K(S)，那么我们就能用枚举的方法穷举一个字符串集合里的内容，然后筛选。


对，就是最没技术含量的那种 for { if (K(S)>=100) } 这种穷举的办法，得到一个符合奇葩程序Q的具体的代码。


而我们已经知道了这个Q不存在，但K(S)如果能写出来，它就存在。


于是通过反证法我们就知道了，唯一可能的情况就是：K(S)是不可能存在的。到这里，柯尔莫罗夫复杂度不可计算已经得证。


看！在怀疑论的严格约束下，我们的知识在增长。


我再举这么一个奇怪的柯尔莫罗夫复杂度的例子是因为仔细思量一下，所谓的学习到某种规律，难道不是一种信息压缩吗？当我们观察到很多次水遇热到一定程度后会变成水蒸气这事儿，观察了一百次得出的领悟，就是把100次的经验描述变成了一句话，这就是压缩。结合IIya之前一次分享活动里对于无监督学习的本质就是压缩信息的总结，有点异曲同工的感受。这个视频值得反复看，里面有另外一种柯尔莫罗夫复杂度不可计算的方法，其实还有很多种办法，有兴趣的可以自己去找，还有通过图灵停机问题扩展证明的，很有趣。


好，如果柯尔莫罗夫复杂度不可计算，也就是信息压缩的极限并不知道，且所有的学习行为本质上都是压缩，所有的科学发现本质上都是“猜想与反驳”对现象经验的浓缩。F=ma 是浓缩现象到四个字符，E=mc^2 也是。那么我们怎么知道我们的总结已经“穷尽天人之究”了呢？或者这个问题的同义句子是：我们怎么知道我们掌握的“规律”就是高度抽象的真理了呢？因为柯尔莫罗夫复杂度不可计算，所以我们不能！于是我可以很有自信的总结到:据我所知，我一无所知:P，这是开玩笑的，事实上我们有可靠的有限的知识。


这又和哥德尔不完备性定律呼应上了，我们构造的逻辑体系必然存在我们不能证明的东西。


再换言之：我们人类，无法掌握真理，永远不能。


而当我们永远无法掌握真理，只能做符合现在观察集内有限的“猜想与反驳”的时候，其意义就是黑天鹅永生。


黑天鹅的意思是我们的经验集合之外的事情永远可能突然出现，这和我们人类无法穷尽了解掌握真理，和信息压缩（学习）的极限不可得，永远在逻辑体系里有不可知的东西，等等等等都是一个意思。


世事无常，唯黑天鹅永生。


但这不是我们束手待毙，毫无作为的同义句。具体到办法上，尤其是交易上，一样不是没有办法，和绕过休谟问题我们用反证法能获取确定的新知识一样，用类似反证法的策略，我们就能获得交易上的可靠性。具体来说，构造的原则就是利用肥尾效应构造不对称策略，“杠铃策略”就是一个典型的例子，在《黑天鹅》里有介绍，更详细的期权版本在《肥尾效应》里有，但说的都很有限。可聪明的你，掌握了这些哲学思想后，可以构造自己的反脆弱的策略，当然前提是足够熟悉衍生品。


这就是我之前通读塔勒布书籍的有限的感想，也许再读还会有新的领悟。

且最最要命的是：你真不知道啥时候会突然飞出一只黑天鹅来。


如果归纳总结规律这件事情本身不可靠，那么因果推断的所有手法是否值得怀疑呢？太阳暴晒过的石头变热了，我们能认为太阳暴晒是石头变热的「因」吗？事实上我们只是看到了两个先后顺序的现象而已，如果有一天出现了暴晒下不变热的石头怎么整？那所有的归纳总结都未必可靠的情况下，甚至整个因果体系是否可靠呢？


不要以为这个就是纯哲学的头脑体操，你所有瞪眼看出的所谓的「K线规律」（别名技术分析）和所有的「基本面因子」全部在归纳法的范畴里。无论你多量化，用多少统计学工具，都在这个范畴里逃不掉。但凡是在过往的历史数据里，无论是量价关系还是基本面信息乃至宏观经济分析，没有不是历史经验归纳法的。而归纳法，本身不可靠。


这个哲学流派有点接近古希腊的「怀疑主义哲学」，值得注意的是怀疑主义并不等价于全然的不可知论。怀疑主义的涵义是：不一定可靠，而彻底的不可知论是严格的你无法知道，这两者是有本质区别的。


远到古希腊的皮浪，近到塔勒布这样的怀疑论者都被人误以为是啥结论都不能下的怪人，毫无意义的偏执狂。《黑天鹅》里塔勒布甚至说有人问他你敢不敢过马路，这都是对怀疑论者的误解，因为你不知道下一秒是否突然有失控的大卡车嘛~~


怀疑论的意思是这些都没有那么值得完全确信，而不是所有的事情都是假的，值得怀疑本身就是一个信息，而非不可知；且已证伪的事情就是一个确凿的信息（很重要），界定上划分的话它是可知的。也就是承认休谟问题的存在并不一定滑向不可知与拔无因果，它们是泾渭分明的两个世界观。


休谟问题只是否认了一般性规律归纳法的可靠性，并不否认所有理性的可靠性。事实上纯的演绎方法的可靠性是可以让我们获得很多认知的，只要反过来就豁然开朗了，接下来我会大量的使用反证法来绕开休谟问题，或者说不用归纳法也是可以获得知识。怀疑论不等于不可知，怀疑论不等于无知识。


人们对于休谟问题的抗拒主要来自于人类自然科学上获得的成就，它确实让工业革命后人类的生存水平极大提高了，可以说工业革命是人类历史上唯一重要的事件。而科学革命就是建立在归纳总结上的，也就是休谟认为不可靠的办法上。科学恰恰就是靠「我见到的天鹅都是白色的，我们认为天鹅是白色的」基础之上的，它切切实实应用了。


嘿嘿~~是不是休谟老哥自己遇上自己的黑天鹅时刻了呢？


关于这个诘难，我见过的最有力的反驳来自于波普尔。他的原话我背不出，大致意思是：如果整个科学都建立在归纳法上，而他违背休谟问题提出的归纳的根本缺陷，我们同样不能先建立科学方法是对的假设，因为那也是基于我们现在的归纳上的，而不应该排除整个科学方法都是有问题的可能。这个纯纯的理性主义，让我觉得波普尔当得起汉传佛典里关于“狮子吼”的比喻，堪称振聋发聩，第一次读之时的心灵震撼记忆犹新。


不要因为自己想要急于知道些什么而迫不及待扑向归纳主义，《黑天鹅》里大量描述了急于知道原因造成的荒诞现象，这些都是正常人类的弱点，有谁希望站在无知的荒原上惶恐不安呢？让我知道点什么，确信点什么吧！啥都不确定是否太让人手足无措了？！处于不安，人类的大脑会大量的使用技能「脑补」，这是极难克服的。


不急不急，咱们来通过飞出黑天鹅的例子来体会如何反向获得确定性的知识，并且用这个知识再反过来推导休谟问题本身为什么蕴含着不可约性。


选的第一个例子是证明「图灵停机问题」的便捷方法。


图灵停机问题是问能否通过一个确定的算法，来判断一段程序是否会结束，也就是是否会陷入死循环。


直觉上，我们写代码的时候有时能瞪眼看出或者一步步Debug推演出一些程序会陷入死循环。这和用一种确定性的算法输入一段程序返回一个Bool返回值是否会陷入死循环是不一样的。也就是我们要写出一个方法，它的输入是一个函数，并且给定运行的参数，具体的算法要返回明确的True或False来判断这个传进去的函数是否能结束。


难道要穷举所有的算法来验证这个问题吗？这是不可能的。我们也不能靠列举出一百万个程序最后都停下来这个事情来归纳出图灵停机问题的答案是所有代码最后都会停下来（归纳的不可靠性）。


但我们只需要通过一个反例就能获取确凿无疑的知识。


假设我们现在确实有一个算法能实现图灵停机判定，这个Halts算法能返回明确的Ture或False来判断一段程序program是否会在对应的输入input的条件下停机。我们假装我们已经有这个算法了。

用这个自相矛盾的例子就能确凿的证明：图灵停机问题的判定函数是不可能存在的，这个Halts算法是绝对不可能写出来的。


我们提出一个假设，并且把这个假设的黑天鹅给飞出来，因为只要有一个黑天鹅的例子就能证伪一个命题，于是我们就能证明这个假设是错的，这样就证明了这个假设的反命题是真的了。这个是我们所有人在数学课堂上都学过的反证法，并不玄奥。上面这个恶作剧式的程序就是图灵停机问题的反证法，非常简洁易懂。


简单归简单，甚至有点玩笑的味道，但想想它的意义：不依赖归纳的真命题出炉了。


构造一个巧妙的悖论是困难的，自指代问题构造起悖论来简直批量可复现。下面我再来用类似的构造自指悖论的反证方法论，得出一个新的知识：柯尔莫罗夫复杂度（Kolmogorov Complexity）不可得。



（事实上这种自指代反证法思路能快捷证明出很多非常困难的问题，包括大名鼎鼎的哥德尔完备性定律和不完备性定律，有兴趣的小伙伴们可以参考视频：https://www.bilibili.com/video/BV1o7421K7pY/?share_source=copy_web&vd_source=ed20ca80cf65eaa0b1224329aaebbb71）



柯尔莫罗夫复杂度这个概念名字里带着复杂两个字，但这个的东西并不复杂，它指能压缩一段字符串的最短长度。比如我给你一串字符串“a@%!fdg8te"，它毫无规律，那么它的最短压缩长度也就是柯尔莫罗夫复杂度约等于它本身。而如果是一个字符串是“abcdefghigklmnopqrstuvwxyz"，那么我可以描述成英语字母表小写，它就被我用描述压缩了。我要还原的话，通过这句话（可以看作一个程序）就能还原。如果一个字符串是个字母表重复10遍，那么这个字符串长达260个字符，我可以通过一个描述“重复10遍英语字母表小写”去压缩这段话。



构造这么一个程序计算K(S)是不可能的，如果存在这么一个函数，那么我们写一个很奇葩的程序Q，要求


“必须大于100行代码才能生成的字符串”


但事实上，就用了一行描述就已经描述了它，明显小于100行，那么这个玩意儿Q就是不存在的。它天生就是悖论的，它自己的定义就让它陷入自我矛盾了。


而如果真的有一个算法K(S)，那么我们就能用枚举的方法穷举一个字符串集合里的内容，然后筛选。


对，就是最没技术含量的那种 for { if (K(S)>=100) } 这种穷举的办法，得到一个符合奇葩程序Q的具体的代码。


而我们已经知道了这个Q不存在，但K(S)如果能写出来，它就存在。


于是通过反证法我们就知道了，唯一可能的情况就是：K(S)是不可能存在的。到这里，柯尔莫罗夫复杂度不可计算已经得证。


看！在怀疑论的严格约束下，我们的知识在增长。


我再举这么一个奇怪的柯尔莫罗夫复杂度的例子是因为仔细思量一下，所谓的学习到某种规律，难道不是一种信息压缩吗？当我们观察到很多次水遇热到一定程度后会变成水蒸气这事儿，观察了一百次得出的领悟，就是把100次的经验描述变成了一句话，这就是压缩。结合IIya之前一次分享活动里对于无监督学习的本质就是压缩信息的总结，有点异曲同工的感受。这个视频值得反复看，里面有另外一种柯尔莫罗夫复杂度不可计算的方法，其实还有很多种办法，有兴趣的可以自己去找，还有通过图灵停机问题扩展证明的，很有趣。


好，如果柯尔莫罗夫复杂度不可计算，也就是信息压缩的极限并不知道，且所有的学习行为本质上都是压缩，所有的科学发现本质上都是“猜想与反驳”对现象经验的浓缩。F=ma 是浓缩现象到四个字符，E=mc^2 也是。那么我们怎么知道我们的总结已经“穷尽天人之究”了呢？或者这个问题的同义句子是：我们怎么知道我们掌握的“规律”就是高度抽象的真理了呢？因为柯尔莫罗夫复杂度不可计算，所以我们不能！于是我可以很有自信的总结到:据我所知，我一无所知:P，这是开玩笑的，事实上我们有可靠的有限的知识。


这又和哥德尔不完备性定律呼应上了，我们构造的逻辑体系必然存在我们不能证明的东西。


再换言之：我们人类，无法掌握真理，永远不能。


而当我们永远无法掌握真理，只能做符合现在观察集内有限的“猜想与反驳”的时候，其意义就是黑天鹅永生。


黑天鹅的意思是我们的经验集合之外的事情永远可能突然出现，这和我们人类无法穷尽了解掌握真理，和信息压缩（学习）的极限不可得，永远在逻辑体系里有不可知的东西，等等等等都是一个意思。


世事无常，唯黑天鹅永生。


但这不是我们束手待毙，毫无作为的同义句。具体到办法上，尤其是交易上，一样不是没有办法，和绕过休谟问题我们用反证法能获取确定的新知识一样，用类似反证法的策略，我们就能获得交易上的可靠性。具体来说，构造的原则就是利用肥尾效应构造不对称策略，“杠铃策略”就是一个典型的例子，在《黑天鹅》里有介绍，更详细的期权版本在《肥尾效应》里有，但说的都很有限。可聪明的你，掌握了这些哲学思想后，可以构造自己的反脆弱的策略，当然前提是足够熟悉衍生品。
