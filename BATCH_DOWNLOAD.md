# 分批下载功能

## 概述

全量下载现在支持分批模式，可以指定每次下载的数量。适合：
- 🕐 时间有限，想分多次完成下载
- 📦 内容量很大，想控制每次下载的数量
- 🔄 定期少量下载，逐步建立本地库

## 使用方法

### 方式 1: 使用 Skill（推荐）

```bash
# 每次下载 10 个帖子
/xz full 10

# 每次下载 20 个帖子
/xz full 20

# 简写方式（直接传数字）
/xz 10
/xz 20

# 全量下载所有（不限制）
/xz full
```

### 方式 2: 使用 npm

```bash
# 每次下载 10 个帖子
npx tsx src/index.ts --mode=full --limit=10

# 每次下载 20 个帖子
npx tsx src/index.ts --mode=full --limit=20

# 全量下载所有（不限制）
npm run sync:full
```

## 使用场景

### 场景 1: 每天下载一点

```bash
# 第1天：下载前10个
/xz 10

# 第2天：继续下载接下来10个
/xz 10

# 第3天：继续...
/xz 10
```

### 场景 2: 时间有限，分批完成

```bash
# 午休时间：下载5个
/xz 5

# 下班后：继续下载5个
/xz 5
```

### 场景 3: 先少量测试，再全量下载

```bash
# 先测试下载2个，看看效果
/xz 2

# 检查 output/ 目录，确认格式正确

# 满意后，下载所有剩余的
/xz full
```

### 场景 4: 控制每次下载的资源占用

```bash
# 每次只下载少量，避免长时间占用浏览器
/xz 5

# 定时运行，逐步完成
```

## 状态管理

分批下载的状态保存在 `.xz/sync_state.json` 的 `synced_topics` 字段中：

```json
{
  "synced_topics": {
    "topic_123": {
      "last_updated": "2026-01-28T10:00:00Z",
      "comment_count": 5
    },
    "topic_124": {
      "last_updated": "2026-01-27T15:00:00Z",
      "comment_count": 3
    }
    // ... 更多已处理的帖子
  }
}
```

每次运行时：
1. 检查 `synced_topics` 中已有的帖子 ID
2. 筛选出未处理的帖子
3. 按限制数量处理
4. 更新 `synced_topics`

## 与断点续传的配合

分批下载和断点续传可以完美配合：

### 抓取阶段中断

```bash
# 开始下载
/xz 10

# 抓取阶段中断（网络问题等）
# 再次运行会从断点继续抓取
/xz 10
```

### 处理阶段中断

```bash
# 开始处理 10 个帖子
/xz 10

# 处理到第 5 个时中断
# 已处理的 5 个已保存状态

# 再次运行，会继续处理剩余的 5 个
/xz 10
```

## 优势

✅ **灵活控制** - 想下载多少就下载多少
✅ **时间可控** - 避免一次性占用太长时间
✅ **资源友好** - 减少单次运行的资源占用
✅ **渐进式** - 可以先少量测试，再批量下载
✅ **可中断** - 随时可以停止，下次继续
✅ **状态清晰** - 随时知道还剩多少未处理

## 注意事项

1. **抓取阶段不受限制**
   - 每次都会抓取全部帖子（用于检查是否有新内容）
   - 只是处理阶段受限制

2. **已处理的不会重复**
   - 已下载的帖子会被跳过
   - 除非有新评论（增量更新时会检测）

3. **清除状态重新开始**
   ```bash
   # 如果想重新下载所有内容
   rm .xz/sync_state.json
   /xz 10
   ```

4. **修改每次数量**
   ```bash
   # 可以随时改变每次的数量
   /xz 5    # 先每次5个
   /xz 10   # 后来改为每次10个
   /xz 20   # 再改为每次20个
   ```

## 与全量下载的区别

| 特性 | 全量下载 | 分批下载 |
|------|---------|---------|
| 命令 | `/xz full` | `/xz full 10` 或 `/xz 10` |
| 处理数量 | 所有未处理的 | 最多 N 个 |
| 完成时间 | 一次性完成 | 需要多次运行 |
| 中途停止 | 可以，下次继续全部 | 可以，下次继续本批 |
| 适用场景 | 首次下载或有充足时间 | 分批下载或时间有限 |

## 示例工作流

### 完整的分批下载流程

```bash
# 第1步：先下载2个测试
/xz 2

# 检查 output/ 目录，确认格式
ls -la output/

# 第2步：每天下载10个
/xz 10  # 第1天
/xz 10  # 第2天
/xz 10  # 第3天
# ...

# 第3步：最后剩余的一次性下载完
/xz full
```

## 推荐使用

- 📱 **首次使用**: 建议先 `/xz 2` 测试
- 🕐 **时间有限**: 使用 `/xz 5` 或 `/xz 10`
- 💪 **有充足时间**: 使用 `/xz full` 一次完成
- 🔄 **定期维护**: 使用 `/xz` 增量更新
